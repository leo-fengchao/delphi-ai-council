# ADR-0014: 多轮辩论 Debate —— 主席对匿名成员定向追问、中枢路由映射、≤3 轮收敛（Phase 8）

- **日期:** 2026-06-04
- **状态:** Accepted（Phase 8 主线）

## 上下文 (Context)
到 Phase 5（ADR-0012）为止，议会流程是「并行作答 → 全员交叉评审 → 主席一次性综合」。主席只凭**一轮**输入（各家初始回答 + 各家评审意见）就直接产出最终结论。实践中这有明显短板：

1. **要点悬而未决**：主席对某些分歧点其实仍不确定，但没有机会向具体某家追问澄清，只能凭现有信息硬下结论。
2. **匿名信息不可深挖**：交叉评审是「各说各话」，没有针对性的一问一答；主席想顺着某条匿名回答继续问「你这个数据来源是什么」时无路可走。

引入**主席对匿名成员的定向追问 + 多轮往返**，能在综合前补齐关键信息，显著提升结论完成度。这正是 README Phase 8 与德尔斐法「多轮独立意见再逐轮收敛」原型的落点。

### 约束回顾
- **C1（ADR-0001）**：零 API、全程驱动原生网页。辩论的每次提问/回答仍是 DOM 注入 + 抽取，**复用各成员已开的标签页**（多轮同标签页串味由 `runtime.ts` 的「基线」机制兜，见 README 运行时健壮性修复 §2）。
- **匿名性（ADR-0006）**：主席全程只见 `Response A/B/C…`，**绝不能知道真实模型身份**。匿名↔真实的映射只存在于插件中枢（Council Page），由中枢负责把主席的追问路由到对应真实模型的标签页。
- **韧性（ADR-0011/0012）**：辩论是新增的多阶段长流程，必须沿用幂等重发、单点降级、崩溃恢复；辩论轮次要落盘以支持刷新/崩溃续跑。
- **额度友好**：辩论是额外的往返开销，**默认关闭**，由用户按需开启（与 Phase 7「深度思考默认开」不同——深度思考是质量基线、几乎零额外往返；辩论是 N×2 轮额外注入抽取）。

## 决策 (Decision)

### 1. 流程位置与触发：stage2 与 stage3 之间，用户开关默认关
- 在 XState 顶层新增 `debate` 态，位于 `stage2_reviewing`（交叉评审）与 `stage3_summarizing`（主席综合）之间。
- Council Page 新增「多轮辩论」开关，**默认关闭**。关闭时 `stage2.onDone` 直接进 `stage3`（老流程零回归）；开启时进 `debate`，收敛后再进 `stage3`。开关状态随 `START` 事件进 `session.enableDebate` 并落盘（恢复时沿用）。

### 2. 稳定的匿名映射 + 初始答案存档（中枢持有映射）
辩论要把「主席对 Response X 的追问」路由到真实模型，必须有一份**全程稳定**的 `anonLabel ↔ adapterId` 映射；且主席综合/辩论引用的「初始回答」必须是 stage1 的原始答案，不能被后续阶段覆盖。

- **存档初始答案**：stage1 完成时，把各家初始回答写入 `session.initialAnswers`（`adapterId → text`）。这修正了一个既有隐患——`runStage2` 会把各腿 `legs[].text` 覆盖为**评审文本**，导致此前 `buildChairpersonPrompt` 的「初始回答」实际取到的是评审文本。改为统一从 `initialAnswers` 取初始答案。
- **稳定匿名映射**：stage1 完成时按 `session.adapterIds` 顺序对「有初始答案」的成员依次编号 `A/B/C…`，存入 `session.anonMap`（`anonLabel → adapterId`）。评审 prompt、主席 prompt、辩论路由全部以此映射为唯一标签来源，保证三阶段标签一致。

### 3. 辩论一轮的 DOM 编排（主席发问 → 中枢路由 → 成员作答 → 回填主席）
每一轮：
1. **主席发问**：在主席标签页注入「辩论提问 prompt」（含匿名初始答案 + 评审意见 + 此前各轮辩论记录），要求主席**要么**用固定可解析格式对指定匿名编号提问，**要么**声明无需追问：
   ```
   ===DEBATE===
   @A: 你对 X 的结论依据是什么？
   @C: 这个数据的来源能给出吗？
   ===/DEBATE===
   ```
   收敛信号：区块内容为 `NONE`（或解析不出任何合法 `@X:` 目标）即视为主席「无需追问」。
2. **中枢解析 + 路由**：解析出 `[{anonLabel, question}]`，经 `anonMap` 反查得到真实 `adapterId`；**剔除指向主席自身的目标**（主席不自我追问）与非法编号。
3. **成员作答**：对每个被点名成员，在其**已开的标签页**里注入「定向追问 prompt」（复用 `driveLegResilient`：幂等、标签页丢失自动重开、内容级失败保留手动重试），并行抽取回答。
4. **回填累积**：把本轮 `[{anonLabel, adapterId, question, answer, status}]` 追加进 `session.debate.rounds` 并落盘；下一轮主席 prompt 带上本轮 Q&A。

### 4. 轮数上限与收敛
- 上限 `MAX_DEBATE_ROUNDS = 3`。达到上限或主席声明 `NONE`/无合法目标即收敛（`session.debate.converged = true`），进入 stage3。
- stage3 的主席综合 prompt **追加完整辩论记录**，让最终结论吸收追问澄清后的信息。

### 5. 韧性与恢复
- `SessionState.status` 增加 `'debate'`；`RESUME` 守卫新增 `status === 'debate' → debate` 分支。
- 辩论按**轮粒度**可恢复：已完整完成的轮直接沿用存档、不重发；某轮内仍有 `pending`/`failed` 目标的，仅重发这些目标（沿用 `driveLegResilient` 的额度友好语义）。再据 `rounds.length` 与 `converged` 决定是否继续生成新轮。
- 单点降级：某成员追问失败（未登录/超时等）不阻断辩论，该目标标 `failed`，主席下一轮/综合时按「该路无回应」继续。

### 6. UI（本阶段最小，详尽 UI 留 Phase 9）
- 新增「多轮辩论」开关（默认关）。
- 新增**辩论轮次时间线**：复用现有卡片/文案风格，逐轮展示「主席 → Response X：追问摘要」「Response X：回复摘要」。不引入组件库；概览方块 / 弹窗详情 / 与状态方块联动留 Phase 9（ADR-0015）。

## 后果 (Consequences)
**好处：**
- 主席能在下结论前对具体匿名回答定向追问澄清，结论完成度与可信度显著提升，契合德尔斐法「逐轮收敛」原型。
- 复用既有韧性设施（`driveLegResilient` / 持久化 / Watchdog）与多轮同标签页「基线」防串味，新增面积可控。
- 顺带修正了「主席综合误取评审文本当初始回答」的既有隐患（`initialAnswers` 存档）。
- 匿名映射收敛为单一真源 `anonMap`，三阶段标签一致，杜绝按 index 重复匿名导致的标签漂移。

**代价 / 妥协：**
- 额外 N×（最多 3）轮注入/抽取，明显增加单场议会耗时与各站额度消耗——故**默认关闭**，由用户为高价值问题按需开启。
- 辩论质量依赖主席模型「按固定格式提问/声明收敛」的指令遵循度；解析不出合法目标时按收敛处理（保守、不卡死），可能错过本应继续的追问。
- 多轮往返放大了「多轮复用同标签页串味」的暴露面，强依赖 `runtime.ts` 基线机制的稳健性；列入 Phase 10 全站回归重点复测项。
- 辩论恢复仅做到轮粒度（轮内仅重发未完成目标），未做「主席发问已发出但抽取前崩溃」的子步精确恢复——此类边界回到该轮重新发问，可接受。
